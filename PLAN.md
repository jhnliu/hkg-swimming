# HKG Swimming Data Platform

## What we have

- 175 PDFs from hkgswimming.org.hk, generated by HY-TEK Meet Manager
- Structured results: swimmer name, ID#, age, club (3-letter code), seed time, finals time, splits
- Events cover long course (LC) and short course (SC), age groups, multiple competition tiers (Group 1/2/3)
- ~10 years of data (roughly 2015–2025)

## Core data model (once parsed to CSV)

| Table        | Key fields                                                        |
|--------------|-------------------------------------------------------------------|
| swimmers     | id, name, gender (derived from event)                             |
| clubs        | code (HVA, WTS, SCA, PSC...), full name                          |
| competitions | id, name, date, venue, course (LC/SC), tier                      |
| events       | competition_id, event_num, gender, distance, stroke, age_group   |
| results      | swimmer_id, event_id, seed_time, finals_time, place, splits[]    |
| standards    | event_id, OQTA, OSTB, WCA, WCB, HKR, JR (qualifying cut times) |

## Feature ideas

### 1. Swimmer Profiles

- Career timeline: every race result across all competitions
- Personal bests by event (LC and SC separately)
- PB progression charts over time
- Age-relative performance (how fast were they at age 14 vs peers?)
- Club history (if they moved clubs)

### 2. Club Analytics

- Club roster by season, total active swimmers
- Club strength by event/stroke — "WTS dominates breaststroke"
- Medal/podium counts per season
- Talent pipeline: how many juniors → senior competitors

### 3. Rising & Falling Stars

- Improvement rate: rank swimmers by % time drop over last 12 months
- Breakthrough alerts: first time beating OST B or OQTA standards
- Consistency score: variance in performance across meets
- Inactivity detection: swimmers who stopped appearing in results
- Leaderboards by age group + event, filterable by season

### 4. Young Swimmer Tracking ("My Swimming")

- Claim your profile (match by name + club + ID)
- Dashboard: PB tracker, where you rank in your age group in HK
- Goal setting: "I want to hit OSTB in 100m Free" → show gap and projected timeline
- Compare with any other swimmer side-by-side
- Shareable progress cards for parents/coaches

### 5. Search & Explore

- Full-text search across swimmers, clubs, competitions
- Event leaderboards: "fastest 100m butterfly, women, age 14-15, long course, all time"
- Head-to-head: compare two swimmers across events they both competed in
- Historical records browser

## Users

- **Primary:** Coaches (clubs and schools) — monitor multiple swimmers, compare across squads, scout talent
- **Secondary:** Swimmers themselves (likely teens) — track own progress, see rankings

## UI/UX Flow

### Key entry points

1. **"Find my swimmer"** — name search → swimmer profile. This is 80% of first visits.
2. **Competition browser** — browse full results from a specific meet (replaces reading the PDF).
3. **Leaderboards / rankings** — "where does my swimmer rank in HK for 100m free, age 13?"

### Pages

1. **Home / Search**
   - Prominent search bar (swimmers, clubs, competitions)
   - Quick links: recent competitions, top leaderboards
   - Language toggle (EN/中文)

2. **Swimmer Profile** `/swimmer/[id]`
   - Header: name, club, age group, gender
   - PB table (LC and SC separately), with HK ranking per event
   - PB progression chart (time series by event)
   - Full race history table (sortable/filterable by event, competition, date)
   - Club history timeline (if moved clubs)

3. **Club Page** `/club/[code]`
   - Club roster (current season), sortable by event strength
   - Aggregate stats: total swimmers, medal counts, podium finishes by season
   - Stroke/event strength breakdown
   - Talent pipeline: junior → senior progression

4. **Competition Page** `/competition/[id]`
   - Full results by event, matching the PDF layout but searchable/filterable
   - Highlight PBs, record swims, standard achievements (J, H, s/S markers)

5. **Leaderboards** `/leaderboards`
   - Filter: event, gender, age group, course (LC/SC), season or all-time
   - Sortable rankings table
   - Coach-friendly: "show me top 10 in 100m breast, boys 13-14, this season"

6. **Head-to-Head** `/compare`
   - Select two swimmers, see side-by-side across shared events
   - Time progression overlay chart

7. **Rising Stars** `/trends`
   - Biggest improvers (% time drop over last 12 months)
   - Breakthrough alerts: swimmers who recently hit qualifying standards
   - Filterable by club, age group, event

### Coach-specific considerations

- All list views should support multi-select / bulk view (a coach wants to see 5 swimmers at once)
- Exportable tables (CSV download) for coaches who still work in spreadsheets
- Dense data display — coaches prefer information density over whitespace

### Language

- Bilingual: English and Traditional Chinese
- URL-based routing: `/en/...` and `/zh/...`
- Swimmer names appear in both languages where available (from PDF data)

## Technical approach

### Tech stack (decided)

- **Framework:** Next.js (App Router) with static export
- **Styling:** Tailwind CSS
- **Charts:** Recharts
- **Search:** Fuse.js (client-side fuzzy search over pre-built index)
- **Data:** Pre-built JSON files at build time (no database for v1)
- **i18n:** Next.js built-in i18n routing (`/en/`, `/zh/`)
- **Hosting:** Vercel or Cloudflare Pages (static, free tier)

### Data pipeline

- PDF → CSV parser (the tricky part — HY-TEK format is consistent, so regex/tabular extraction should work well)
- Dedup swimmers across competitions (ID# field is key — it's the HK Swimming registration number)
- Normalize club codes, handle name variations
- Output: JSON files matching the data model, consumed by Next.js at build time

### Build order

1. Define JSON data schema (what the UI expects)
2. Generate realistic mock data
3. Build UI skeleton against mock data
4. Build PDF parser separately (outputs same schema)
5. Plug in real data, validate

## Key parsing challenges to watch for

- Swimmer names have inconsistent formatting (English name order varies)
- The ID# field (e.g. 30192 B, 20540 IAZ) is the best dedup key
- Split times appear on a second line under each result
- Some results have suffixes like J (junior record), H (HK record), s/S (standard achieved)
- Need to infer gender from event header ("Men"/"Women"), not from individual rows

## What makes this valuable

No one is doing this for HK swimming right now. The official site just hosts raw PDFs — there's no way to search by swimmer, track progress over time, or compare across meets. Parents and coaches currently do this manually in spreadsheets. Even a basic searchable database would be a significant upgrade.
